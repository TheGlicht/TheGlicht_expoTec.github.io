What is?

Is a method for testing software programs. In this technique, the data entered into the software to be tested is divided into partitions of equal size. From each data partition, a test case is needed. The different test cases should test the software classes continuously.



In this technique, the input data is divided into partitions or equivalence classes, data that produce the same result are grouped in the same class, i.e., all members of a partition are expected to be processed in the same way.

  • Each input data must belong to only one partition or equivalence class.
  • Valid values are those that the system is supposed to accept. The partitions that contain the valid values are called "valid equivalence partition".
  • Similar to the previous point, values that are rejected by a system are called invalid values and the partitions containing them are known as "invalid equivalence partition".
